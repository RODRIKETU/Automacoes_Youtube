🎯 Visão Geral

A API do YouTube Automation System permite integração completa com o sistema de criação automática de vídeos. Todas as respostas são em formato JSON e utilizam códigos de status HTTP padrão.

Base URL

http://localhost:3000

Formato de Resposta

Todas as respostas seguem o padrão JSON com os seguintes campos:

{ "success": boolean, "data": object, "message": string, "timestamp": string }

🛠 Endpoints Disponíveis

GET Listar Temas Disponíveis

GET /api/themes

Retorna todos os temas disponíveis para criação de vídeos.

Resposta 200 OK

{ "success": true, "data": [ { "id": 1, "nome": "Contos de Terror Urbano", "descricao": "Histórias assombradas em cenários urbanos", "cor": "#800020", "icone": "building", "tags": ["terror", "urbano", "suspense"] } ] }

GET Estatísticas do Sistema

GET /api/stats

Retorna estatísticas completas do sistema incluindo métricas por tema, arquivos e logs.

Resposta 200 OK

{ "success": true, "data": { "geral": { "total_projetos": 15, "concluidos": 12, "com_erro": 2, "publicados": 10, "em_andamento": 1 }, "por_tema": [...], "por_dia": [...], "arquivos": [...], "erros_recentes": [...] } }

POST Criar Novo Projeto

POST /api/projects

Inicia a criação de um novo vídeo com o tema especificado.

Parâmetros

Parâmetro Tipo Obrigatório Descrição
tema_id integer Sim ID do tema selecionado
titulo_personalizado string Não Título personalizado para o vídeo

Exemplo de Requisição

{ "tema_id": 1, "titulo_personalizado": "Minha História de Terror" }

Resposta 200 OK

{ "success": true, "data": { "projeto_id": "proj_123456", "status": "iniciado", "tema": "Contos de Terror Urbano", "created_at": "2025-08-20T21:30:00Z" }, "message": "Projeto criado com sucesso" }

GET Status do Projeto

GET /api/projects/{projeto_id}/status

Consulta o status atual de um projeto específico.

Parâmetros de URL

Parâmetro Tipo Descrição
projeto_id string ID único do projeto

Resposta 200 OK

{ "success": true, "data": { "projeto_id": "proj_123456", "status": "processando", "progresso": 65, "etapa_atual": "gerando_audio", "tempo_estimado": "5 minutos", "arquivos_gerados": ["script.txt", "audio.mp3"] } }

GET Logs do Sistema

GET /api/logs

Retorna os logs recentes do sistema com filtros opcionais.

Query Parameters

Parâmetro Tipo Padrão Descrição
limit integer 50 Número máximo de logs
level string all Nível do log (error, warning, info)
projeto_id string - Filtrar por projeto específico

Exemplo

GET /api/logs?limit=10&level=error

📋 Códigos de Status HTTP

Código Status Descrição
200 OK Requisição executada com sucesso
400 Bad Request Parâmetros inválidos ou em falta
404 Not Found Recurso não encontrado
500 Internal Server Error Erro interno do servidor

🔐 Autenticação

Atualmente a API é executada em ambiente local sem autenticação. Para ambiente de produção, implemente:

  • 🔑 API Keys - Headers personalizados
  • 🎫 JWT Tokens - Autenticação por token
  • 🛡️ Rate Limiting - Controle de requisições
  • 🔒 HTTPS - Comunicação criptografada

💡 Exemplos de Uso

JavaScript/Fetch

// Criar novo projeto const response = await fetch('http://localhost:3000/api/projects', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ tema_id: 1, titulo_personalizado: 'Meu Vídeo Épico' }) }); const data = await response.json(); console.log('Projeto criado:', data.data.projeto_id);

Python/Requests

import requests # Listar temas response = requests.get('http://localhost:3000/api/themes') temas = response.json() # Criar projeto novo_projeto = { 'tema_id': 1, 'titulo_personalizado': 'Vídeo Python' } response = requests.post( 'http://localhost:3000/api/projects', json=novo_projeto ) print(f"Status: {response.status_code}") print(f"Projeto ID: {response.json()['data']['projeto_id']}")

cURL

# Listar estatísticas curl -X GET http://localhost:3000/api/stats # Criar novo projeto curl -X POST http://localhost:3000/api/projects \ -H "Content-Type: application/json" \ -d '{"tema_id": 1, "titulo_personalizado": "Teste cURL"}'
